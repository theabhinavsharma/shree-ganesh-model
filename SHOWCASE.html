The Architecture

5 ML engines, parallel + cross-validated

A name only ships if it appears in ≥2 engines' top-15 AND survives contamination QC AND operator-pump detection AND RSI < 80 filter. Most names die in the cascade.

# How the cross-engine vote works
def is_actionable(symbol):
    in_engines = sum(symbol in top_15(e) for e in ENGINES)  # count of top-15 hits
    if in_engines < 2:                  return False         # need cross-validation
    if contaminated(symbol):            return False         # unrecorded -30%+ drop
    if operator_pump(symbol):           return False         # 3+ consecutive UCs
    if rsi(symbol) > 80:                return False         # chase zone
    if adv_cr(symbol) < 2.0:            return False         # illiquid
    if score_5pct_15d(symbol) < 0.55:   return False         # below floor
    return True

The Binding Rules

The constitution that keeps us honest

Anthropic-grade structured prompts and binding operating rules. Every commit, trade, and live prediction passes these gates. Not aspirational — actually enforced in code and pytest.

  • §1.1 — Honesty about uncertainty
    A claim of "85% hit rate" must cite a parquet path, year range, sample size, or it doesn't ship. In-sample and prospective are never confused.
  • §1.2 — Pre-commit to falsification
    Every new claim runs through devils_advocate.py BEFORE shipping. When new evidence contradicts a prior claim, lead with the contradiction. Never smooth over.
  • §1.3 — Strict walk-forward
    Train ≤ year_X-1. Calibrate on year_X-1 only. Test on year_X. Calibrators never fit on test data, even if the metric "looks better" the other way.
  • §1.4 — Multiple-testing discipline
    If N hypotheses tested, significance bar is α/N (Bonferroni). "Best of N" reporting is forbidden. Report median, range, and count tested.
  • §1.5 — Survive first, win second
    The -30% annualised drawdown floor is sacred. When the discipline cascade returns 0 fire-day signals, the answer is cash. Never force a trade to look productive.
  • §1.6 — Living calibration ledger
    Every mistake corrected goes into logs/calibration_corrections.jsonl with date, claim, correction, and the parquet path that exposed it. Past mistakes are not repeated.
  • §1.7 — Reproducibility = publishability
    A claim that cannot be reproduced from a fresh clone + the daily pipeline is not publishable, no matter how strong the number.
